Ich entwickle ein SaaS-/Freemium-Produkt namens „Data Health Management“ für Microsoft Dynamics 365 Business Central (BC365).

Ziel:
Unternehmen sollen nur eine BC-Extension installieren müssen und danach automatisch eine kostenlose Vorabanalyse ihrer Datenqualität erhalten. Die Vorabanalyse liefert einen DataScore und erste Kennzahlen. Eine kostenpflichtige Detailanalyse (einmalig oder im Abo) soll später zusätzliche Statistiken und konkrete Handlungsempfehlungen liefern.

Wichtige Produktlogik:
- Kostenlos:
  - Installation der BC-Extension
  - QuickScan
  - DataScore
  - erste Kennzahlen / Top Issues
- Kostenpflichtig:
  - DeepScan
  - detaillierte Statistiken
  - konkrete Handlungsempfehlungen
  - Verlauf / Monitoring / Abo-Modell

Datenschutzansatz:
- möglichst nur aggregierte Kennzahlen übertragen
- keine Rohdaten, keine sensiblen Geschäftsdaten standardmäßig exportieren
- BC-Extension sammelt Metriken
- externer Scoring-Service berechnet Score und liefert Ergebnisse zurück

Architektur:
- BC Extension in AL
- Backend mit Python + FastAPI
- Docker / Docker Compose
- BC Cloud Sandbox zum Testen
- lokales Backend aktuell über ngrok öffentlich erreichbar

Aktueller technischer Stand:
- Git-Repository mit Struktur für backend, bc-extension und docs
- FastAPI Backend läuft
- Endpoints vorhanden:
  - GET /health
  - POST /tenant/register
  - POST /scan/quick
- Docker Compose funktioniert lokal
- BC-Extension ist in einer BC365 Cloud Sandbox installiert
- Verbindung zur Cloud-Sandbox funktioniert
- HttpClient Requests sind für die Extension erlaubt
- API Base URL in BC Setup ist aktuell eine ngrok-URL
- Tenant-Registrierung funktioniert
- QuickScan funktioniert end-to-end
- Ergebnis in BC aktuell per Message Box, z. B.:
  - DataScore: 90
  - Top Issues: 10 Artikel ohne Kategorie

Aktuelle AL-Funktionalität:
- Setup-Tabelle vorhanden
- Setup-Page vorhanden
- API Client vorhanden
- QuickScan sendet aktuell erste 5 Kennzahlen an /scan/quick:
  - customers_total
  - customers_missing_postcode
  - customers_missing_payment_terms
  - items_total
  - items_missing_category
